Understanding lab.conf

What is lab.conf?#

lab.conf is the file used for configuring your Netkit lab. This should be present in the lab directory.

The main purpose of this config is to specify network configuration. We can define the interfaces of a machine in the lab, and specify which collision domains these interfaces are connected to.

This guide follows on from the introduction, so it is recommended you have a look at this first.

Defining Network Interfaces#

To define an interface within lab.conf we use the following format:

MACHINENAME[INTERFACE_NUMBER]=NETWORK

Where:

  • MACHINENAME is the name of the machine in the lab.
  • INTERFACE_NUMBER is the number of the interface you want to set. This should be a non-negative integer (0 or higher) It makes sense to start your interfaces from 0, and count up from there if you need multiple interfaces.
  • NETWORK is normally the collision domain - this is a virtual switch, and can be specified with a simple string. More advanced options are available such as tapout interfaces. See lab.conf for more information.

For example, we want machines a and r to each have have an interface connected to the same virtual switch. We can use the following in our lab.conf

a[0]=lanX
r[1]=lanX

This means machine a will have an interface eth0 which is connected to the lanX collision domain, and machine r will have an interface eth1 which is also connected to this collision domain. These machines will therefore be able to communicate as if they are connected to the same switch.

Assigning Machine Memory#

The default amount of RAM assigned to a Netkit VM is 128MB. This can be changed in lab.conf to specify the amount of memory by machine. We use the following format:

MACHINENAME[mem]=NUM

Where:

  • MACHINENAME is the name of the machine in the lab.
  • NUM is the amount of RAM in MegaBytes
  • mem is the literal string "mem"

For example, if we want to give 512MB of memory to machine r:

r[mem]=512

Setting Lab Information#

In lab.conf you can set lab information variables, which will be shown when starting a lab. This can be useful for quickly identifying the purpose of a lab, or for giving your information as an author when distributing the lab, which can give you credit and allow users of the lab to contact you if needed.

The following options are available:

  • LAB_DESCRIPTION - a basic description of the lab
  • LAB_VERSION - the version of the lab
  • LAB_AUTHOR - the author(s) of the lab
  • LAB_EMAIL - contact(s) for lab authors
  • LAB_WEB - URLs for useful resources

Example lab.conf section:

LAB_DESCRIPTION="Netkit lab to explain XYZ concept"
LAB_VERSION="V1"
LAB_AUTHOR="John Doe"
LAB_EMAIL="johndoe@example.com"
LAB_WEB="https://github.com/johndoe/netkit-lab-xyz"

Advanced#

For more information on what can be set in lab.conf, check the manpages here or run man lab.conf.
